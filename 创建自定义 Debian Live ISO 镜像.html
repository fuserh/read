前言

参考 Will Haley 大佬的这篇文章:
https://www.willhaley.com/blog/custom-debian-live-environment/
为了简化操作稍微魔改了一下, 移除了 UEFI 支持, 仅支持 BIOS 传统模式引导, 如需 UEFI 引导请参考原文.

最终实现效果是生成一个可以引导的 ISO 镜像, 其中包含了预装好的 Docker, vim, openssh-server 等应用.

实现过程

· 用 VMware Workstation 先启动一台虚机, 正常安装好 Debian Bullseye 系统, 然后安装后面会用到的工具(均在 root 用户环境执行):

# 切换使用清华大学的镜像rm /etc/apt/source.list && echo "deb https://mirrors.tuna.tsinghua.edu.cn/debian/ bullseye main contrib non-free" > /etc/apt/source.list# 安装所需工具apt updateapt install -y debootstrap squashfs-tools xorriso isolinux grub-pc-bin mtools dosfstools# 创建中转的文件夹mkdir /LIVE_BOOT# 使用 debootstrap 安装基础系统文件debootstrap --arch=amd64 --variant=minbase bullseye /LIVE_BOOT/chroot https://mirrors.tuna.tsinghua.edu.cn/debian/# 使用 chroot 切换根目录chroot /LIVE_BOOT/chroot

· 下面操作在 chroot 环境中进行

# 修改 hostnameecho "debian-live" > /etc/hostname# 为目标系统安装所需的软件包apt install -y linux-image-amd64 live-boot systemd-sysv network-manager net-tools iproute2 iputils-ping zip unzip curl vim openssh-server# 安装 Dockerexport DOWNLOAD_URL="https://mirrors.tuna.tsinghua.edu.cn/docker-ce" && curl -fsSL https://get.docker.com/ | sh# 清理 apt 缓存apt clean# 设置 root 用户密码passwd root# 修改 SSH 配置允许 root 远程登陆sed -i "s/\#PermitRootLogin prohibit-password/PermitRootLogin yes/g" /etc/ssh/sshd_config# 退出 chroot 环境exit

· 继续在原系统进行

# 创建系统引导和临时文件夹mkdir -p /LIVE_BOOT/{staging/{isolinux,live},tmp}# 压缩打包成 squashfs 文件系统

mksquashfs /LIVE_BOOT/chroot /LIVE_BOOT/staging/live/filesystem.squashfs -e boot# 复制引导文件到目标 LIVE 系统cp /LIVE_BOOT/chroot/boot/vmlinuz-* /LIVE_BOOT/staging/live/vmlinuzcp /LIVE_BOOT/chroot/boot/initrd.img-* /LIVE_BOOT/staging/live/initrdcp /usr/lib/ISOLINUX/isolinux.bin "/LIVE_BOOT/staging/isolinux/"cp /usr/lib/syslinux/modules/bios/* "/LIVE_BOOT/staging/isolinux/"# 创建 grub 引导菜单配置文件cat <<'EOF' >/LIVE_BOOT/staging/isolinux/isolinux.cfg

UI vesamenu.c32

 

MENU TITLE Boot Menu

DEFAULT linux

TIMEOUT 3

MENU RESOLUTION 1024 768

 

LABEL linux

  MENU LABEL Debian Live [BIOS/ISOLINUX]

  MENU DEFAULT

  KERNEL /live/vmlinuz

  APPEND initrd=/live/initrd boot=live

 

LABEL linux

  MENU LABEL Debian Live [BIOS/ISOLINUX] (nomodeset)

  MENU DEFAULT

  KERNEL /live/vmlinuz

  APPEND initrd=/live/initrd boot=live nomodeset

EOF# 创建 ISO 镜像

xorriso \

    -as mkisofs \

    -iso-level 3 \

    -o "/LIVE_BOOT/debian-custom.iso" \

    -full-iso9660-filenames \

    -volid "DEBLIVE" \

    --mbr-force-bootable -partition_offset 16 \

    -joliet -joliet-long -rational-rock \

    -isohybrid-mbr /usr/lib/ISOLINUX/isohdpfx.bin \

    -eltorito-boot \

        isolinux/isolinux.bin \

        -no-emul-boot \

        -boot-load-size 4 \

        -boot-info-table \

        --eltorito-catalog isolinux/isolinux.cat \

    "/LIVE_BOOT/staging"

· 至此就可以得到 /LIVE_BOOT/debian-custom.iso 镜像文件

测试镜像

创建一台新的虚机, 使用 debian-custom.iso 进行引导, Bingo!
